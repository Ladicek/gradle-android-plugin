This package contains everything needed to work with ADB. Everything in this package is public except of the Adb class.

Public API

  • ConnectedDevices provides a way to enumerate all connected Android devices.
  • Device contains relevant informations about an Android device (its serial number and wheter it is an emulator or not). ConnectedDevices returns a List of these.
  • AdbExec is a Gradle task that allows to run adb command with user-defined parameters. Note that the device parameter (-d, -e or -s) should not be specified. If you want to specify a particular device, AdbExec has a device property for that. See also Specifying a device below.
  • SelectedDevice is only used by AdbExec. It isn't useful for users.

Specifying a device

AdbExec tasks reads a device from (in order of evaluation):

  • device system property (-Ddevice=...).
  • adb.device.arg system property (-Dadb.device.arg=...). This is deprecated.
  • device property of itself.

Value of the device system property and the device property of AdbExec task can be:

  • emulator: the single connected emulator. It is an error if no emulator or 2 or more emulators are connected.
  • usb: the single USB connected device. It is an error if no device or 2 or more devices are connected.
  • <serial number>: any other string, which is considered to be a serial number. It is an error if no device with such serial number is connected.

Adb class

Direct calls to Adb class are limited (on purpose):

  • AndroidPlugin calls Adb.init
  • ConnectedDevices calls Adb.devices

It could be reimplemented by e.g. parsing adb devices output and only direct callers would have to be changed. Thus the public API of this package should be pretty solid.
